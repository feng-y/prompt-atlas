# Inbox

这里保存尚未处理的真实 Prompt、聊天摘录、文章线索或执行结果。它是队列，不是长期知识。

建议文件名：`YYYY-MM-DD-short-topic.md`。最低内容：

- 原始 Prompt 或材料；
- 目标模型 / 工具；
- 真实任务背景（只写会改变判断的部分）；
- 发送前最不确定的一点；
- 若已执行：结果、返修轮次、验证证据。

## 下一步

- 只需暂存或仍不确定价值：保留在 `inbox/`。
- 会成为案例、原则或任务判断依据：按 [source record](../templates/source-record.md) 注册到 `sources/`，然后按 [AGENTS.md](../AGENTS.md) 执行 ingest。
- 形成完整 review：进入 `prompt-pairs/<ability>/`。
- 只有跨多个真实案例重复验证的规律，才能进入 `principles/` 或 `patterns/`。
