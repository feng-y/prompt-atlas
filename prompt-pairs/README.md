# Prompt Pairs

案例按主要训练能力归档；一个案例可以同时带多个标签，但只保留一个主落点。

每个案例记录一次真实 Prompt 的提升过程：`Current Prompt → Improvement → Minimal Rewrite → Why It Is Better → Final Prompt`。重点是让读者看清这次改版改变了什么，不要求在每个 case 中展开完整的 Prompt review。

Foundation、Task Contract、Evidence、自治边界、工具路由和停止规则属于 Guide 的内部分析方法。case 只保留理解本次提升所必需的分析；页面末尾的 `Final Prompt` 是唯一可直接复制发送的成稿。

已有案例可以保留较完整的历史分析结构，不为统一形式批量重写。后续只在案例过重、提升主线不清或需要实质更新时渐进收缩。
